# Tools

Small HTML/CSS/JS tools you build during Week 5. Everything here runs in the browser — no server, no build step, no dependencies. Open the file, it works.

---

## Why this folder exists

Because a retrospective timer you built yourself beats an app you pay for. Because a team randomizer tuned to how *your* team works is more useful than a generic one. Because being able to *make* a tool in 15 minutes changes how you facilitate.

---

## What lives here

By the end of Week 5 you'll have built at least:

- a **retro timer** (counts down, makes a sound)
- a **team randomizer** (shuffles names or pairs)
- a **silent voting** tool (dot voting without crosstalk)
- an **engagement temperature check** (quick 1–5 pulse)

Each tool is a single `.html` file. Double-click to open. Use it in your next ceremony.

---

## Rules

- **Client-side only.** No backend. No API keys. No npm install.
- **One file per tool.** HTML, CSS, and JS in the same file is fine.
- **Real use first.** Don't polish until you've used it with a team at least once.
- **Publish in Week 8.** These will appear on your [GitHub Pages](https://pages.github.com/) site.
